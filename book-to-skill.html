六步 SOP 详解

书 → 纯文本

拿到整本书干净的 .txt，图表丢了没关系，文字逻辑在就行。
三种方式：Z-Library（直接选 TXT）、Calibre（免费转换）、python-docx（技术向）。
关键：整本直接喂，不要提前摘要——摘要把框架精度丢光。

全局结构地图

用 Prompt 让 AI 通读全书，输出：核心论点 + 命名框架/原则/技法 + 逻辑关系 + 作者语气。

五类台账提炼

按「框架 / 原则 / 技法 / 反模式 / 作者声音」五类提炼。每条给：名字 + 定义 + 何时用 + 原书表述。框架保留原作者命名，不改写成通用说法。标注优先级 P0/P1/P2。

生成 SKILL.md

框架少（1-3 个）→ 单文件；框架多（10+）→ 入口文件 + 分文件懒加载。
description 黄金法则：写满触发场景，不能说「一个XX的skill」，要说「当用户需要...时调用」

自检 + 试调用（不可跳过）

召回测试：准备 5 句用户日常说话，看能否命中 skill。任何一句没命中 → 回补 description 触发词。
调用测试：用一个真实问题测试，看输出是可执行步骤还是空话。

多模型 PK（可选）

同一份台账丢给不同模型各生成一版，对比提炼精度。推荐组合：Claude Opus 4.8 + DeepSeek V4 + Gemini 2.5 Pro。

2
Prompt 工具箱（直接复制）
Prompt 1：全局结构地图
第二步 · 通读全书
这是《[书名]》全文。先不要摘要。 通读全书，给我一张全局结构地图： - 核心论点是什么？ - 围绕它有哪些命名框架/原则/技法/反模式？ - 它们之间什么逻辑关系？ - 作者的语气有什么辨识度？ 我想用它做一个能帮我 [你的目的] 的 skill。
Prompt 2：五类台账提炼
第三步 · 结构化提炼
按这 5 类提炼这本书：框架 / 原则 / 技法 / 反模式 / 作者声音。 每条给：名字 + 一句话定义 + 何时用 + 原书的精确表述。 框架要保留作者的原始命名，别改写成通用说法。 哪些是核心必进、哪些是边角可省？标注优先级 P0/P1/P2。
Prompt 3：生成 SKILL.md
第四步 · 产出 Skill 文件
按这份台账生成标准的 SKILL.md 文件。 要求： 1. description 写满触发场景 2. 保留原作者所有命名框架 3. 每条指令必须可执行 4. 包含具体示例 5. 语言风格自然、去 AI 味
description 黄金法则对照：
❌ 错误：「一个非暴力沟通的 skill」
✅ 正确：「当用户需要提意见但不显得像指责、化解冲突、和家人/同事沟通敏感话题时调用」
五类台账定义优先级
框架有步骤、有顺序的结构化方法论P0 核心
原则指导性的行动准则P0 核心
技法具体可执行的技巧P1 重要
反模式常见的错误做法P1 重要
作者声音独特语气/风格/隐喻/金句P0 保留

多模型PK：Claude Opus 4.8 版本对比融合后新增商业应用和教养应用模块。
案例 2：《好课像大片》
13.8万字符Word文档客户定制

五类台账：框架 9 个 · 原则 9 条 · 技法 10 条 · 反模式 8 条
客户专属模块：课程诊断检查表（8个问题查任何一课）+ 调整旧课 6 步操作流程
第五步结果：召回测试 5/5 命中，调用测试给出完整诊断和重构建议。
