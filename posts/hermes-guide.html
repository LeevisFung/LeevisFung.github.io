1. Hermes 是什么？

Hermes 是一个开源的 AI Agent 框架，可以理解为一个「能装插件的 ChatGPT」。它的核心优势：

  • 免费开源 — 不用每月花 $200 订阅 Claude Max
  • Skill 生态 — 200+ 社区技能，想干啥装啥
  • 本地运行 — 数据不离开你的电脑
  • 记忆能力 — 装上 hindsight 后能记住所有对话
  • 支持国内模型 — 可以接 DeepSeek、通义千问、Kimi 等国内 API，完全不用翻墙
💡 适合谁？

产品经理、运营、独立开发者、内容创作者 — 任何想让 AI 帮你干活但不想花钱订阅的人。尤其适合国内用户，搭配 DeepSeek API 使用体验极好。

2. 安装环境准备

你需要准备以下环境（15 分钟搞定）：

2.1 安装 Node.js

# macOS / Linux brew install node # Windows # 去 https://nodejs.org 下载 LTS 版本安装 # 验证安装 node --version # 需要 >= 18.x

2.2 安装 Git（可选但推荐）

# macOS brew install git # Windows: 去 https://git-scm.com 下载
💡 国内用户网络加速

如果 npm install 很慢，先设置国内镜像：
npm config set registry https://registry.npmmirror.com

3. 获取 API Key（国内用户必看）

Hermes 支持接入几乎所有主流 AI 模型。以下按推荐优先级排列，国内用户首选 DeepSeek。

🥇 DeepSeek 强烈推荐

注册送 $5 国内直连 速度极快 超便宜

国内最强开源模型，V4-Pro 参数 1.6T，中文能力顶级，OpenAI API 格式完全兼容。

获取步骤：

  1. 访问 platform.deepseek.com
  2. 用手机号或邮箱注册（国内手机号直接收验证码）
  3. 登录后点击左侧 「API Keys」 → 点「创建 API Key」
  4. 复制保存 key（只显示一次，格式为 sk-xxxxxxxx）

免费额度：注册即送 $5，每天 50 次免费调用。日常轻度使用基本不花钱。

价格：V4-Flash 输入 $0.14/百万 token，输出 $0.28/百万 token — 是 GPT-4 的 1/100。

API 地址：https://api.deepseek.com | 模型名：deepseek-chat (V4) / deepseek-reasoner (R1推理)

# Hermes 配置 DeepSeek hermes config set api-key sk-your-deepseek-key hermes config set base-url https://api.deepseek.com hermes config set model deepseek-chat

🥈 月之暗面 Kimi 推荐

有免费额度 国内直连 长上下文

Kimi 以超长上下文著称（128K），适合分析长篇文档、论文、合同等场景。

获取步骤：

  1. 访问 platform.moonshot.cn
  2. 注册后进入控制台 → 「API Keys」→ 创建密钥
  3. 新用户赠送 15 元体验额度

API 地址：https://api.moonshot.cn/v1 | 模型名：moonshot-v1-8k / moonshot-v1-128k

# Hermes 配置 Kimi hermes config set api-key sk-your-kimi-key hermes config set base-url https://api.moonshot.cn/v1 hermes config set model moonshot-v1-128k

🥉 通义千问 Qwen（阿里云百炼）

10万次/月免费 国内直连 企业级稳定

阿里云百炼平台，企业版每月 10 万次免费调用，稳定性高，有丰富的 SDK 支持。

获取步骤：

  1. 访问 bailian.console.aliyun.com
  2. 用阿里云账号登录 → 开通「百炼」服务
  3. 点击右上角头像 → 「API-KEY 管理」→ 创建 API Key

API 地址：https://dashscope.aliyuncs.com/compatible-mode/v1 | 模型名：qwen-plus / qwen-max

智谱 ChatGLM

赠送额度 国内直连

清华系，GLM 模型在多模态和代码能力上有优势。

获取步骤：

  1. 访问 open.bigmodel.cn
  2. 注册后进入「API Keys」页面创建密钥
  3. 新用户赠送一定额度

API 地址：https://open.bigmodel.cn/api/paas/v4 | 模型名：glm-4 / glm-4-flash

豆包（火山引擎）

超低价格 国内直连

字节跳动旗下，Doubao 模型价格极低，适合大批量调用场景。

获取步骤：

  1. 访问 console.volcengine.com/ark
  2. 注册火山引擎账号 → 开通「豆包大模型」服务
  3. 在「API Key 管理」创建密钥

API 地址：https://ark.cn-beijing.volces.com/api/v3 | 模型名：doubao-pro-32k

OpenAI / Anthropic（海外直连）

性能最强

如果你有海外网络条件，也可以直接接 GPT-4o 或 Claude 模型：

  • OpenAI — platform.openai.com/api-keys
  • Anthropic — console.anthropic.com

长文档：Kimi（处理大型文档、合同、论文）
重活：OpenAI GPT-4o（有海外网络时，复杂推理场景用）

4. Hermes 安装步骤

# 第1步：安装 Hermes CLI npm install -g @hermes-ai/cli # 第2步：初始化项目 hermes init my-agent cd my-agent # 第3步：配置 API Key（DeepSeek 示例） hermes config set api-key sk-your-deepseek-key hermes config set base-url https://api.deepseek.com hermes config set model deepseek-chat # 第4步：启动！ hermes start

看到 "Hermes is running!" 就说明安装成功了 🎉

⚠️ 首次启动要点

第一次启动时会下载一些依赖包，可能等 1-2 分钟。如果卡住了可以 Ctrl+C 后重新 hermes start。

6. 5个真实工作流（完整操作步骤）

上篇笔记只是简介，这里是完整操作步骤：

工作流1：自动写周报

前提：hindsight 已运行至少 3 天，积累足够的对话记忆。

# 1. 确保 hindsight 已安装并运行了3天以上 hermes chat # 2. 让它总结你这一周做了什么 > 帮我总结这周的工作内容，按项目分组 # 3. 生成正式周报 > 按公司周报模板格式输出，每个项目包含：目标、进展、下周计划 # 4. 用 avoid-ai-writing 去味（可选） > 用 avoid-ai-writing 过一遍刚才的周报
⚠️ 踩坑

头 3 天 hindsight 啥都记不住，因为还没积累足够的对话。别急，坚持用就行。另外，它不会主动记录你在终端里打的命令，需要你在聊天里提一句「我刚跑了 git push」。

工作流2：一句话出设计稿

hermes chat > 用 open-design 帮我做一个极简风格的登录页 # Hermes 会自动调用 open-design 生成 UI # 输出文件在 ./output/login-page.html # 如果要调整： > 把按钮颜色改成品牌色 #FF6D00，背景加个模糊渐变
⚠️ 踩坑

字体有时会乱码，手动在 HTML 里换成本地字体就行。复杂布局（比如 Dashboard）生成质量一般，简单页面效果最好。

工作流3：画流程图不用手

hermes chat > 画一个用户注册流程图，包含手机号验证和邮箱验证两条路径 # 自动生成 draw.io XML 文件 # 下载到 draw.io 打开即可编辑 # 进阶：画系统架构图 > 画一个微服务架构图，包含网关、用户服务、订单服务、数据库
⚠️ 踩坑

超过 20 个节点的复杂图会丢连线，建议拆成多个小图分别生成。另外生成的 XML 文件需要 draw.io 桌面版打开，网页版可能加载失败。

工作流4：多 Agent 同时跑

# 启动 mission-control hermes install skill mission-control hermes mission start # 在 GUI 里创建3个 Agent： # Agent1: 写文案（用 avoid-ai-writing 做后处理） # Agent2: 做配图（用 open-design 生成） # Agent3: 查数据（用 browser-browse 爬网页） # 在控制台看3个 Agent 同时工作
⚠️ 踩坑

同时开超过 4 个 Agent 电脑会卡，特别是做图那个比较吃资源。API 调用量也会成倍增加，注意费用。

工作流5：AI 写的稿过一遍去味

hermes chat > 帮我写一篇小红书种草笔记，主题是"程序员省时间神器" # 写完后让 avoid-ai-writing 处理 > 用 avoid-ai-writing 过一遍刚才那篇，去掉 AI 味 # 处理完直接复制发布
⚠️ 踩坑

过度去味会变成口水话，跑一遍就够，别跑两遍。实测数据显示：去味前收藏率 ~2%，去味后能到 ~8%。

8. 常见踩坑 & 解决

❌ 安装时报 "permission denied"

sudo npm install -g @hermes-ai/cli # macOS/Linux # Windows: 用管理员身份打开终端

❌ Skill 安装后不生效

hermes skill list # 看看是不是显示 installed hermes restart # 重启 Hermes

❌ API 报 429 错误（限速）

# DeepSeek 每天 50 次免费调用，超了就等明天 # 或者换个模型继续用： hermes config set model deepseek-chat # 换回 V4（V3 可能已经限速）

❌ 两个 Skill 打架了

# 用 SkillClaw 自动检测冲突 hermes install skill skillclaw hermes skillclaw check

❌ hindsight 说"没有记忆"

头 3 天正常，它需要积累对话数据。多用它聊天就行。如果一周后还不行，检查 hintight 是否在 skill list 里显示为 "active"。

❌ DeepSeek API 连接超时

# 检查 base-url 是否正确 hermes config get base-url # 正确值为：https://api.deepseek.com # 不要加 /v1 后缀（Hermes 会自动加）

9. 进阶：自己写 Skill

Hermes 的 Skill 本质就是一个 Markdown 文件 + 可选的 JS 脚本：

# 创建 Skill 目录 mkdir -p ~/.hermes/skills/my-skill cd ~/.hermes/skills/my-skill # 创建 SKILL.md（必须） cat > SKILL.md << 'EOF' --- name: my-skill description: 我的自定义技能 trigger: 当用户说"帮我XXX"时触发 --- # My Skill 你是一个 XXX 专家。当用户请求 XXX 时： 1. 先了解需求 2. 给出方案 3. 执行并验证 EOF # 重新加载 Skill hermes skill reload
💡 技巧

写 Skill 最重要的不是代码，是 trigger 触发条件 写清楚。触发条件越精准，Skill 越不容易被误调用。建议参考社区高星 Skill 的 SKILL.md 写法。
